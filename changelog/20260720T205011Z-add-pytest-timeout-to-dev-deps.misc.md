Added `pytest-timeout` as a dev dependency and configured per-test (`timeout = 30`) and session-level (`session_timeout = 900`) timeouts in `[tool.pytest.ini_options]`. Limits are set with large safety margins above the measured worst-case single test (~4 s) and full-suite wall time (~75 s). On a timeout hit the logged timeout value and offending test id make it straightforward to distinguish a true deadlock from a slow-but-healthy test.
